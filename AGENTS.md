# Codex Night-Run Safety Policy

本文件用于约束 Codex 在本项目中的自动执行行为，目标是：
- 允许高效率自动化执行（包括联网）
- 同时保证文件系统与代码资产安全

## 1) 授权范围

- 工作目录白名单：`/Users/jiama/Desktop/compileFIle/codex_proj`
- Codex 仅允许在上述目录内进行读写、创建、删除、执行操作。
- 不得主动修改白名单外路径（包括用户家目录其他位置、系统目录、外接盘目录）。

## 2) 网络权限

- 网络访问：**允许全部外网访问**（不做域名限制）。
- 允许下载代码、依赖、模型、数据集与文档。
- 涉及账户凭据时，不得在日志中明文输出 token、密码、私钥。

## 3) 命令白名单（默认允许）

- Git 类：`git clone`、`git pull`、`git fetch`、`git checkout -b`、`git status`、`git log`
- 环境类：`conda create`、`conda run`、`pip install`
- 运行类：`python`、`bash scripts/*`、`pytest`、`npm/yarn/pnpm`（如项目用到）
- 检索类：`rg`、`ls`、`cat`、`sed`、`find`

## 4) 高风险操作（必须二次确认）

以下操作即使在白名单目录内，也必须先询问并获得明确确认：
- 大范围删除：`rm -rf`（尤其是递归删除目录）
- 覆盖式清理：`git reset --hard`、`git checkout -- .`、批量覆盖写入
- 系统级操作：`sudo`、改系统配置、改权限到全局路径
- 影响不可逆的数据迁移或重写历史的操作

## 5) 文件系统安全约束

- 禁止触达系统关键路径：`/System`、`/Library`、`/usr`、`/private`（与项目无关部分）
- 禁止修改 SSH/GPG/Keychain 等凭据相关文件
- 仅在项目内新建临时文件、缓存目录、日志目录

## 6) Git 安全约束

- 默认在当前分支工作，建议优先新建任务分支。
- 不得擅自改写历史（rebase/force push）除非用户明确要求。
- 不得回滚或覆盖用户未授权变更。

## 7) 日志与可追溯

- 所有关键步骤应可复现（命令可回放）。
- 关键执行结果应写入控制台并可汇总说明。
- 若生成脚本，优先放到 `scripts/` 目录。

## 8) 失败与异常处理

- 若遇权限/网络/依赖问题，应先在项目内尝试自恢复。
- 如需突破安全边界，必须暂停并请求用户确认。
- 出现潜在破坏性风险时，立即停止并报告。

## 9) 默认执行模式

- 允许尽量自动化完成端到端任务（含联网下载）。
- 在不触发“高风险操作”前提下，可连续执行。
- 每次任务结束需给出：改动文件、执行摘要、可复跑命令。
